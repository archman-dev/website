## Overview

Different stakeholders need different perspectives: executives want business context; developers need technical detail. Views and viewpoints provide structured ways to represent architecture for clarity.

### Models Covered

1. **4+1 Model** – Logical, process, physical, development views + scenarios.

2. **C4 Model** – Context, Container, Component, Code levels with progressive detail.
